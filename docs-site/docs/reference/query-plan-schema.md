# QueryPlan JSON Schema

The `QueryPlan` IR is the discriminated union of `SelectPlan`,
`AskPlan`, and `ConstructPlan`. The full JSON Schema is generated
from the runtime models with `pydantic.TypeAdapter`.

## Where to get the schema

Three options:

1. **Static asset on this site** —
   [`/schema/query-plan.schema.json`](/schema/query-plan.schema.json).
2. **MCP resource** — `graph://query-plan/schema`.
3. **Programmatically** —
   ```python
   from pydantic import TypeAdapter
   from graph_mcp.models import QueryPlan

   schema = TypeAdapter(QueryPlan).json_schema()
   ```

The static asset is regenerated by
`scripts/generate_docs_reference.py`, so it always matches the
runtime IR.

:::tip
The full schema is large — over a hundred class definitions. For
reading, prefer the curated excerpts below.
:::

## Top-level discriminator

```json
{
  "kind": "select" | "ask" | "construct"
}
```

Each shape includes:

| Field | Type | Notes |
| --- | --- | --- |
| `prefixes` | `Prefix[]` | Optional. Built-ins are seeded by the validator. |
| `where` | `Pattern[]` | Required. |

## Curated examples

### Simple SELECT

```json
{
  "kind": "select",
  "prefixes": [{"prefix": "ex", "iri": "http://example.org/"}],
  "projection": [{"var": {"name": "person"}}],
  "where": [
    {
      "kind": "triple",
      "subject": {"kind": "var", "name": "person"},
      "predicate": {"kind": "prefixed_name", "prefix": "ex", "local": "worksFor"},
      "object": {"kind": "prefixed_name", "prefix": "ex", "local": "Acme"}
    }
  ],
  "limit": 50
}
```

### OPTIONAL

```json
{
  "kind": "optional",
  "patterns": [
    {
      "kind": "triple",
      "subject": {"kind": "var", "name": "person"},
      "predicate": {"kind": "prefixed_name", "prefix": "rdfs", "local": "label"},
      "object": {"kind": "var", "name": "label"}
    }
  ]
}
```

### UNION

```json
{
  "kind": "union",
  "branches": [
    [{"kind": "triple", "subject": {"kind": "var", "name": "x"},
       "predicate": {"kind": "prefixed_name", "prefix": "ex", "local": "knows"},
       "object": {"kind": "var", "name": "y"}}],
    [{"kind": "triple", "subject": {"kind": "var", "name": "x"},
       "predicate": {"kind": "prefixed_name", "prefix": "ex", "local": "worksFor"},
       "object": {"kind": "var", "name": "y"}}]
  ]
}
```

### FILTER NOT EXISTS

```json
{
  "kind": "filter",
  "expression": {
    "kind": "not_exists",
    "patterns": [
      {
        "kind": "triple",
        "subject": {"kind": "var", "name": "person"},
        "predicate": {"kind": "prefixed_name", "prefix": "rdfs", "local": "label"},
        "object": {"kind": "var", "name": "label"}
      }
    ]
  }
}
```

### Property path (one-or-more)

```json
{
  "kind": "triple",
  "subject": {"kind": "var", "name": "x"},
  "predicate": {
    "kind": "one_or_more",
    "operand": {
      "kind": "term",
      "iri": {"kind": "prefixed_name", "prefix": "ex", "local": "knows"}
    }
  },
  "object": {"kind": "var", "name": "y"}
}
```

### Aggregate query

```json
{
  "kind": "select",
  "projection": [
    {"var": {"name": "company"}},
    {
      "expression": {
        "kind": "aggregate",
        "function": "count",
        "expression": null
      },
      "alias": {"name": "n"}
    }
  ],
  "where": [
    {
      "kind": "triple",
      "subject": {"kind": "var", "name": "person"},
      "predicate": {"kind": "prefixed_name", "prefix": "ex", "local": "worksFor"},
      "object": {"kind": "var", "name": "company"}
    }
  ],
  "group_by": [{"kind": "var", "name": "company"}],
  "order_by": [{"expression": {"kind": "var", "name": "n"}, "descending": true}]
}
```

### Subquery (top-N per group)

```json
{
  "kind": "subquery",
  "select": {
    "kind": "select",
    "projection": [{"var": {"name": "person"}}],
    "where": [
      {
        "kind": "triple",
        "subject": {"kind": "var", "name": "person"},
        "predicate": {"kind": "prefixed_name", "prefix": "ex", "local": "worksFor"},
        "object": {"kind": "var", "name": "company"}
      }
    ],
    "limit": 5
  }
}
```

### Named graph

```json
{
  "kind": "graph",
  "graph": {"kind": "iri", "value": "https://graphs.example.com/g1"},
  "patterns": [
    {
      "kind": "triple",
      "subject": {"kind": "var", "name": "s"},
      "predicate": {"kind": "var", "name": "p"},
      "object": {"kind": "var", "name": "o"}
    }
  ]
}
```

When `GRAPH_MCP_ALLOWED_GRAPHS` is non-empty, the validator requires
the graph IRI to be on the allowlist (or, for `GRAPH ?g`, a sibling
`VALUES ?g` constraining it to allowlisted IRIs).
