# QueryPlan IR

The `QueryPlan` IR is the contract between the LLM and the rest of the
system. It is a strict, recursive Pydantic v2 schema; every other
layer (validator, renderer, executor) treats a successfully parsed
plan as ground truth.

The full schema is exported as
[`/schema/query-plan.schema.json`](/schema/query-plan.schema.json),
generated by `scripts/generate_docs_reference.py`.

## Top-level shape

`QueryPlan` is a discriminated union on `kind`:

| Kind | Class | Purpose |
| --- | --- | --- |
| `select` | `SelectPlan` | row-shaped result |
| `ask` | `AskPlan` | boolean result |
| `construct` | `ConstructPlan` | RDF graph result |

All three share `prefixes: list[Prefix]` and `where: list[Pattern]`.

## Term level

The smallest building blocks live in `graph_mcp/models/iri.py`:

| Class | Notes |
| --- | --- |
| `Var` | A SPARQL variable; matches `^[A-Za-z_][A-Za-z0-9_]{0,127}$`. |
| `Iri` | Absolute IRI. Validates against `ABSOLUTE_IRI_REGEX`; rejects relative IRIs. |
| `PrefixedName` | `prefix:local`. Resolved against the plan's prefix map (with built-in defaults seeded by the validator). |
| `LiteralValue` | Typed RDF literal; exactly one of `datatype` or `lang` may be set. |
| `Prefix` | A `PREFIX prefix: <iri>` declaration. |

`RdfTerm` is the discriminated union accepted in subject/object
position: `Var | Iri | PrefixedName | LiteralValue`.

## Patterns

Patterns nest inside `where`. The full set:

```text
TriplePattern   GroupPattern   OptionalPattern   UnionPattern
MinusPattern    FilterPattern  BindPattern       ValuesPattern
GraphPattern    ServicePattern SubqueryPattern
```

Each is a separate Pydantic model with a `kind` discriminator. The
union type is `Pattern`. The validator walks `Pattern` recursively;
the renderer dispatches on `kind`.

`TriplePattern` predicates may be a `Var`, an `Iri | PrefixedName`,
**or** a `PropertyPath`. Subjects and objects cannot be property
paths — the discriminated `RdfTerm` enforces that statically.

## Property paths

Property paths are their own discriminated union (`PropertyPath`):

```text
PropertyPathTerm        — atomic predicate, optional ^ inverse
PropertyPathInverse     — ^path
PropertyPathSeq         — a/b/c
PropertyPathAlt         — a|b
PropertyPathZeroOrMore  — path*
PropertyPathOneOrMore   — path+
PropertyPathZeroOrOne   — path?
```

The validator measures path complexity with a tree-walk and rejects
paths over `policy.max_property_path_complexity`. Unbounded paths
(`*`, `+`) are disabled unless `policy.allow_unbounded_paths=true`.

## Expressions

Expressions live inside `FilterPattern`, `BindPattern`, projection
items, `OrderClause`, `GROUP BY`, `HAVING`, and aggregate inputs.

Allowed primitives are gated by allowlists in
`graph_mcp/models/_ir.py`:

| Allowlist | Members |
| --- | --- |
| `ALLOWED_FUNCTIONS` | `str`, `lcase`, `ucase`, `strlen`, `contains`, `strstarts`, `strends`, `substr`, `concat`, `isiri`, `isuri`, `isblank`, `isliteral`, `isnumeric`, `datatype`, `lang`, `abs`, `ceil`, `floor`, `round`, `if`, `coalesce`, `uri`, `iri` |
| `ALLOWED_BINARY_OPS` | `=`, `!=`, `<`, `<=`, `>`, `>=`, `+`, `-`, `*`, `/`, `&&`, `\|\|` |
| `ALLOWED_UNARY_OPS` | `-`, `+` |
| `ALLOWED_AGGREGATES` | `count`, `sum`, `avg`, `min`, `max`, `sample`, `group_concat` |
| `ALLOWED_DATETIME_ACCESSORS` | `year`, `month`, `day`, `hours`, `minutes`, `seconds`, `now` |

Anything outside the allowlists is rejected at IR construction time
with a `ValidationError` (Pydantic), before the validator runs.

## Strict Pydantic configuration

Every IR class uses `_StrictBase`:

```python
class _StrictBase(BaseModel):
    model_config = ConfigDict(extra="forbid", strict=True)
```

`extra="forbid"` rejects unknown fields. `strict=True` rejects loose
type coercion (no `"123"` becoming `123`). Attempts to smuggle JSON
that "looks close" fail at boundary parsing.

## Recursive resolution

Pydantic v2 resolves forward refs by inspecting module globals.
Because `Expression`, `Pattern`, and `PropertyPath` live in the same
module (`_ir.py`), every forward reference resolves without ambient
globals tricks. After defining all the classes, the module calls
`_rebuild_recursive_models()` once, passing an explicit
`_REBUILD_NAMESPACE` for resilience.

The shape is verified by:

- `tests/test_models.py` — basic IR construction;
- `tests/test_import_robustness.py` — import under multiple
  `PYTHONHASHSEED` values, plus a re-import smoke test.

## Built-in prefixes

`graph_mcp/models/literals.py` declares `DEFAULT_PREFIXES`:

```python
DEFAULT_PREFIXES = {
    "rdf":  "http://www.w3.org/1999/02/22-rdf-syntax-ns#",
    "rdfs": "http://www.w3.org/2000/01/rdf-schema#",
    "owl":  "http://www.w3.org/2002/07/owl#",
    "xsd":  "http://www.w3.org/2001/XMLSchema#",
    "skos": "http://www.w3.org/2004/02/skos/core#",
    "dct":  "http://purl.org/dc/terms/",
    "foaf": "http://xmlns.com/foaf/0.1/",
}
```

Validator and renderer both seed their prefix maps from this
dictionary, so a plan using `rdf:type` does not need to redeclare the
`rdf:` prefix. Plans **may** redeclare a built-in to its canonical
IRI; redeclaring it to a different IRI fails with
`default_prefix_override` unless
`policy.allow_default_prefix_override=true`.
